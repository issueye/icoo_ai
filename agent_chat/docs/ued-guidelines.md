# agent_chat UED 规范

## 1. 设计定位

`agent_chat` 采用桌面 IM 工作台风格，参考 QQ 桌面端的浅蓝界面结构：左侧窄导航、中间会话列表、右侧聊天工作区。整体气质应轻、亮、克制，优先服务长时间阅读、扫描、切换会话和处理 Agent 工作流。

关键词：

- 桌面聊天
- 浅蓝水感
- 高密度信息
- 低噪音边界
- Agent 工作台

## 2. 布局规范

### 2.1 全局结构

```text
AppShell
├── Global Header：40px，无边框窗口拖拽区和全局操作
└── App Body
    ├── Nav Rail：60-64px
    ├── Conversation Sidebar：228-304px
    └── Chat Workspace：自适应
```

### 2.2 尺寸 token

| 名称 | 值 | 用途 |
| --- | --- | --- |
| `--nav-rail-width` | `64px` | 左侧主导航 |
| `--conversation-sidebar-width` | `288px` | 会话列表 |
| `--global-header-height` | `40px` | 无边框窗口顶部栏 |
| `--chat-header-height` | `64px` | 聊天顶部栏 |
| `--composer-height` | `168px` | 输入区基础高度 |
| `--chat-content-max-width` | `1180px` | 最大化后的聊天内容轨道 |
| `--message-max-width` | `680px` | 消息最大宽度 |

### 2.3 间距

- 页面结构间距使用 `12px / 16px / 20px`。
- 会话项内部间距使用 `10px / 12px`。
- 消息流垂直间距使用 `12px - 16px`。
- 工具卡和审批卡内部间距使用 `12px`。

## 4. 字体规范

- 中文 UI 优先：`Microsoft YaHei UI`, `Microsoft YaHei`, `PingFang SC`, `Noto Sans SC`。
- 英文和数字沿用系统 sans。
- 等宽内容：`Cascadia Code`, `JetBrains Mono`, `Consolas`。
- 不使用负字距，所有文字 `letter-spacing: 0`。

| 场景 | 字号 / 行高 | 字重 |
| --- | --- | --- |
| 顶部会话标题 | `15px / 22px` | `700` |
| 会话标题 | `13px / 18px` | `600` |
| 会话摘要 | `12px / 17px` | `400` |
| 消息正文 | `14px / 22px` | `400` |
| 工具元信息 | `12px / 18px` | `400` |

## 5. 组件规范

### 5.1 左侧导航

- 宽度固定 `64px`。
- 图标按钮使用 `40px` 点击区域。
- 当前入口使用白色半透明底和轻微阴影。
- 红点只用于待审批、未读或异常。

### 5.1.1 全局 Header

- 应用使用无边框窗口，顶部 `Global Header` 负责拖拽、品牌识别和窗口操作。
- Header 高度固定 `40px`，左侧为可拖拽区域，右侧所有按钮必须是不可拖拽区域。
- 窗口按钮使用图标：最小化、最大化/还原、关闭。
- 关闭按钮 hover 使用危险色，其余按钮使用白色半透明 hover。

### 5.2 会话列表

- 搜索框高度 `36px`，浅蓝半透明背景。
- 当前会话使用高饱和青蓝背景。
- 会话项高度不固定，但应保持紧凑，头像 `40px`。
- `sess_` 使用蓝色标识，`subsess_` 使用紫色标识。

### 5.3 聊天区

- 顶部栏高度 `64px`。
- 消息区域背景使用浅蓝，不使用大面积白色。
- 消息流和输入区必须共享同一个居中内容轨道，最大化后不能出现消息靠左、输入框居中的错位。
- Agent 消息白底，用户消息青蓝底。
- 系统、工具、审批事件作为轻量卡片，不展示完整大输出。

### 5.4 输入区

- 输入区固定在底部，背景略白。
- 工具栏使用图标按钮，不用大段说明。
- 主发送按钮使用 `--qq-active`。
- 禁用态降低透明度，不改变布局。

## 6. 圆角与阴影

- 普通按钮、输入框、会话项：`8px`。
- 消息气泡：`8px`，靠近头像侧可收窄到 `4px`。
- 工具卡、审批卡：`8px`。
- 避免多层卡片嵌套。
- 阴影只用于浮起态，默认更多依赖颜色层次和边界线。

## 7. 交互规范

- hover 只改变背景或轻微位移，不做夸张动效。
- transition 时长 `140ms - 180ms`。
- focus 必须显示蓝色 ring。
- 会话切换、审批、发送都应有即时反馈。
- 长任务状态使用文本 + 色点，不能只靠颜色表达。

## 8. CSS 使用规则

- 优先使用 `src/styles/globals.css` 中的 `qq-*` 组件类。
- 新组件需要先复用 token，再新增语义类。
- 禁止在组件中继续散落大面积 `bg-[#...]` 硬编码颜色。
- 允许少量 Tailwind 布局类，如 `flex`、`grid`、`gap-*`、`truncate`。
