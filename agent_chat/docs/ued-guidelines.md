# agent_chat UED 设计规范

## 1. 布局减层原则

- 页面优先使用扁平结构，避免不必要的包裹层。
- 单页结构建议为：`header + content`，其中 `content` 仅保留一个主布局容器。
- 同类信息尽量使用统一卡片/面板样式，不重复嵌套相同视觉语义的容器。
- 表单类页面优先使用“列表总览 + 明细编辑”模式，减少长表单直接堆叠在主视图。

## 2. 渠道管理模块规范

- 渠道管理是一级目录模块，不作为设置页子项。
- 渠道支持动态新增与删除，允许同类型多实例（例如多个 QQ、多个飞书）。
- 渠道主视图使用卡片网格展示，每张卡片仅保留：
  - 渠道名称与说明
  - 渠道类型
  - 启用状态切换
  - 关键字段摘要（App ID / Secret / Token / Webhook）
  - `编辑明细` 操作入口
- 渠道新增必须使用弹窗流程完成，避免在主视图直接插入大表单。
- 渠道详细配置必须通过弹窗编辑，不在卡片中直接展开大段输入区。
- 明细弹窗包含完整可编辑字段：
  - `渠道类型`
  - `渠道名称`
  - `App ID`
  - `App Secret`
  - `Bot Token`
  - `Webhook URL`
- 启用状态和明细编辑都应即时保存并给出明确反馈（成功/失败 toast）。

## 3. 交互与可读性

- 敏感字段在卡片中必须脱敏显示。
- 长字段（例如 webhook）在卡片中使用截断摘要，完整值在弹窗中查看与编辑。
- 弹窗需支持点击遮罩关闭、右上角关闭按钮、取消按钮和保存按钮。
